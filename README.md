Instead of following this description exactly, we took some of the major ideas and combined some of our own. The exoplanet archive is still utilized and is turned into a pandas data frame that stores all of the information organized into columns and rows. This is where some of the similarities start to dissolve, but all of the necessary elements are still integrated such as pandas, astropy, skyfield, and much more. The database is organized and called by the title of what is needed like planet_names or planet_types. 

The goal of our code is to create a fully functioning streamlit that takes a closer look at exoplanets in an interactive way. Before continuing with the code, it is extremely important to double check that the exoplanet catalog is downloaded as a CSV into the users computer. Without this Csv file, the code will not run correctly or smoothly. Upon opening up the streamlit, a user is met with a website that is separated into 3 different columns. The first column is labeled Exoplanet Selection. This is the start of the magic; the user will pick from somewhere around 6,000 exoplanets. After making a selection, there will be an image that portrays the type of exoplanet--terrestrial, gas giant, Neptune-like, super earth, or unknown. Underneath this image there will be an additional column of information that looks at all the observations included in the CSV file for that specific exoplanet. The second column offers additional information on the chosen exoplanet. This is where the LLM comes in to answer questions, and after answering this question, there is a JSON block that give the exoplanet name, type, and the distance if this information is given in the csv file. The chat with the AI does not clear itself, so when you switch to a different exoplanet, previous conversations can still be viewed should the user want to look back on information. The final and third column shows a visual representation of where the exoplanet lies based on RA and DEC. There are additional stars that are on this visual, with larger stars being the brightest, and the smaller ones being dimmer. The large red dot shows where the exoplanet is. 

The code itself is broken up into multiple documents:  main.ipynb, streamlit_exoplanet.py, exoplanet.csv, jpeg/jpg images, and the llm_setup.py. The main.ipynb is used mostly as a way to double check that all of the documents are correctly existing in the paths. Although it doesn't hold significance to the actual stream lit itself, it is important to check as why there may be an error in the streamlit_exoplanet.py. It also allows for multiple people to work on the code at the same time, as one person can work in main and someone else has the ability to work in the streamlit. The star of the show of course is the streamlit_exoplanet.py. This is where all of the visuals, and the definitions of the code is created. A .py file cannot have a a .ipynb file imported so each variable or anything that is created in a .ipynb file must be copied and pasted into the .py file to ensure that everything is defined and works smoothly. Since all of the main code is stored here, it is organized by column after important definitions and imports are made. The LLM is given prompts to answer using the CSV file and utilize the option chosen in the first columns to answer questions. The exoplanet.csv file contains all of the information is given about the exoplanets, and the data is sorted through by taking out the first 96 rows to ensure that everything runs smoothly. Going back to the streamlit_exoplanet.py, the first column uses the exoplanet.csv file to create all of the options for the planet names. Since there are multiple observations of one planet, the data has to get sorted through, so the planet names only show up one time. The Jpeg and jpg images are used to give visual representation of what a certain type of exoplanet looks like based on how it is characterized. These images are from the NASA website and are considered to be an artistic visual of the planets. The llm_setup.py is exactly what it sounds like it is. This file simply defines the llm, the version it is, and the characterization of it. For this project it is given the rule of a researcher.
